# 常见问题

> 更新日期：2026-07-22（中止后立即释放发送门禁；僵尸 executing 可自动恢复）

排障速查与本地数据说明。安装问题也可对照 [快速开始](../getting-started.md)。

## 安装与启动

**Q. Windows 提示 SmartScreen / macOS 提示无法打开？**  
安装包默认未代码签名，属预期。按系统提示允许运行即可。更多见 [apps/desktop/README.md](../../apps/desktop/README.md)。

**Q. macOS 提示「SuperVideoGenerator 已损坏，无法打开。你应该将它移到废纸篓」？**  
多数情况下**不是文件坏了**，而是 Gatekeeper 把从网上下载的未公证应用标了隔离属性（quarantine）。按下面做一次即可：

1. 确认已把 `.app` 拖到「应用程序」（或记下实际路径）。
2. 打开「终端」，执行（路径按你的安装位置改）：

```bash
xattr -cr /Applications/SuperVideoGenerator.app
```

3. 再双击打开。若仍被拦，到 **系统设置 → 隐私与安全性**，在被拦截提示旁点 **「仍要打开」**。

仍打不开时再核对：是否下载了与芯片匹配的包（Apple Silicon 用 arm64，Intel 用 x64），以及是否从官方 [Releases](https://github.com/GodyuFF/SuperVideoGenerator/releases) 下载。

**Q. 启动后立刻报错 `Cannot find module './openExternal.cjs'`？**  
这是旧安装包漏打该模块导致的。请下载包含修复的新版本（`electron-builder.yml` 已将 `openExternal.cjs` 列入打包白名单）覆盖安装；开发壳不受影响。

**Q. 开发壳启动失败？**  
确认本机有 Python 3.11+、Node.js 18+，已创建 `.venv` 并安装 `requirements.txt` 与 `apps/web` 的 npm 依赖。用 `launch-desktop.bat` 可看控制台日志。

## AI 与对话

**Q. 一直提示「去配置 AI」？**  
打开 **「AI 配置」**，完成 LLM（及故事书所需的图片）并 **「保存并返回对话」**。见 [AI 配置](02-ai-config.md)。

**Q. 输入框灰了、发不了消息，也不能编辑剧本？**  
常见原因：

1. **AI 正在执行**（顶栏显示「执行中…」、按钮变为「中止执行」）——此时输入与手动编辑会暂时锁定；等执行完成，或点 **「中止执行」**。中止后后端会立刻释放发送门禁；若仍锁死，刷新页面。
2. **有确认卡片待处理**（输入框显示「等待确认…」）——先点卡片上的 **「继续」** / **「重新生成」** / **「中止」**。
3. **从其他剧本切换过来后仍锁定**——刷新页面，或点一次 **「中止执行」**（若后端已无任务会自动解锁）。
4. **剧本状态卡在 executing（无真实任务）**——重启 API 或再次发送时会自动恢复为 failed 并允许对话。

**Q. 为什么视频风格不能改了？**  
该剧本首次发送后风格已锁定。请新建剧本再选其他风格。见 [视频风格与模式](06-modes.md)。

**Q. 目标模式和普通模式差在哪？**  
目标模式少弹确认、更自主；首次建议关闭。见 [对话与执行计划](03-chat-and-plan.md)。

**Q. AI 视频一直失败？**  
检查 **「视频」** Tab 是否启用并填写有效 Key；确认当前剧本风格确实需要视频生成。

## 导出与数据

**Q. 导出提示媒体未加载？**  
等待剪辑助手中素材加载完成后再点 **「导出」**。

**Q. 项目文件在哪？会不会上传？**  
默认留在本机。常见路径：

| 路径 | 内容 |
|------|------|
| `data/` | 项目、剧本、媒体、对话、AI 配置 |
| `.env` | 环境变量与 API Key（若使用） |
| `.venv/` | 本机 Python 虚拟环境（不入库） |

请勿将上述含密钥或媒体的内容提交到 Git。

**Q. 换电脑如何迁移？**  
自行备份 `data/`（及若使用的 `.env`），在新环境安装应用后放回对应位置；仍需保证依赖与 FFmpeg 可用。
